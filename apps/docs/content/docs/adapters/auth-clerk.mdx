# Clerk auth adapter

`@magnet-cms/adapter-auth-clerk` supplies an `AuthMagnetProvider` so Magnet validates **Clerk-issued JWTs**. Sign-in, sign-up, and refresh run in Clerk; your API accepts `Authorization: Bearer <token>` (or the `__session` cookie).

## Installation

```bash
bun add @magnet-cms/adapter-auth-clerk @clerk/backend
```

## Registration

Include a **database** adapter in the same provider array.

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'
import { ClerkAuthStrategy } from '@magnet-cms/adapter-auth-clerk'

@Module({
  imports: [
    MagnetModule.forRoot([
      MongooseDatabaseAdapter.forRoot(),
      ClerkAuthStrategy.forRoot({
        secretKey: process.env.CLERK_SECRET_KEY,
        jwtKey: process.env.CLERK_JWT_KEY,
        authorizedParties: ['https://your-app.example.com'],
        defaultRole: 'user',
      }),
    ]),
  ],
})
export class AppModule {}
```

With only environment variables set (`CLERK_SECRET_KEY`, etc.), `ClerkAuthStrategy.forRoot()` is enough:

```typescript
ClerkAuthStrategy.forRoot()
```

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `CLERK_SECRET_KEY` | Yes | Secret key for `verifyToken` and Clerk APIs |
| `CLERK_PUBLISHABLE_KEY` | No | Publishable key (frontend / docs) |
| `CLERK_JWT_KEY` | No | PEM public key for networkless JWT verification |
