# Redis cache adapter

`@magnet-cms/adapter-cache-redis` provides a `CacheMagnetProvider` for `MagnetModule.forRoot([...])`, backed by [ioredis](https://github.com/redis/ioredis). Compatible with Redis 6+ and Dragonfly.

## Installation

```bash
bun add @magnet-cms/adapter-cache-redis ioredis
```

## Registration

Always include a **database** provider in the same array.

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'
import { RedisCacheAdapter } from '@magnet-cms/adapter-cache-redis'

@Module({
  imports: [
    MagnetModule.forRoot([
      MongooseDatabaseAdapter.forRoot(),
      RedisCacheAdapter.forRoot(),
    ]),
  ],
})
export class AppModule {}
```

Explicit URL:

```typescript
RedisCacheAdapter.forRoot({ url: process.env.REDIS_URL, keyPrefix: 'myapp:' })
```

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `CACHE_REDIS_URL` | No | Preferred Redis URL for cache |
| `REDIS_URL` | No | Fallback if `CACHE_REDIS_URL` is unset |

If neither URL is set, the adapter falls back to `host` / `port` options (default `localhost:6379`).
