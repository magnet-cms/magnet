Magnet composes **providers** you pass to `MagnetModule.forRoot([...])`. The database adapter is required; other adapters add auth, media storage, secrets, email, cache, or GraphQL.

See [Configuration](/docs/getting-started/configuration) for how providers and global options fit together.

## Database adapters

| Adapter | Package | Databases | Features |
|---------|---------|-----------|----------|
| Mongoose | `@magnet-cms/adapter-db-mongoose` | MongoDB | Transactions, full-text search, geospatial, change streams |
| Drizzle | `@magnet-cms/adapter-db-drizzle` | PostgreSQL, MySQL, SQLite | Transactions, full-text search, migrations |

### MongoDB (Mongoose)

```bash
bun add @magnet-cms/adapter-db-mongoose mongoose
```

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'

@Module({
  imports: [
    MagnetModule.forRoot([MongooseDatabaseAdapter.forRoot()]),
  ],
})
export class AppModule {}
```

### PostgreSQL (Drizzle)

```bash
bun add @magnet-cms/adapter-db-drizzle drizzle-orm pg
```

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { DrizzleDatabaseAdapter } from '@magnet-cms/adapter-db-drizzle'

@Module({
  imports: [
    MagnetModule.forRoot(
      [
        DrizzleDatabaseAdapter.forRoot({
          dialect: 'postgresql',
          driver: 'pg',
        }),
      ],
    ),
  ],
})
export class AppModule {}
```

`DATABASE_URL`, `MONGODB_URI`, and `JWT_SECRET` are typically read from `process.env` when you use adapter defaults.

## Database adapter contract

All database adapters implement the `DatabaseAdapter` abstract class from `@magnet-cms/common` (`connect`, `forFeature`, `model`, `supports`, `getCapabilities`, etc.). For the full surface area (features, capabilities, `Model`, query builder), refer to the types and JSDoc in **`@magnet-cms/common`** and the [Mongoose](/docs/adapters/mongoose) / [Drizzle](/docs/adapters/drizzle) guides.

## Injection tokens

```typescript
import { getModelToken, getSchemaToken, getAdapterToken } from '@magnet-cms/common'

const modelToken = getModelToken('User')
const schemaToken = getSchemaToken('User')
const adapterToken = getAdapterToken()
```

## Custom database adapters

To support a new database:

1. Subclass `DatabaseAdapter` in `@magnet-cms/common`.
2. Implement `connect`, `forFeature`, `model`, `token`, `onModuleDestroy`, `supports`, and `getCapabilities`.
3. Return a `DatabaseMagnetProvider` from a static `forRoot()` factory (see existing adapters).
