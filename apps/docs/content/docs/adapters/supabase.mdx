# Supabase Adapter

Supabase integration for Magnet CMS is provided by three separate adapter packages:

1. **Auth** — `@magnet-cms/adapter-auth-supabase` — Replace Magnet's built-in JWT auth with Supabase Auth
2. **Storage** — `@magnet-cms/adapter-storage-supabase` — Use Supabase Storage for media uploads
3. **Vault** — `@magnet-cms/adapter-vault-supabase` — Use Supabase Vault (pgsodium) for secrets

For database connectivity with a Supabase PostgreSQL database, use the [Drizzle adapter](/docs/adapters/drizzle) with a Supabase connection string.

## Installation

```bash
# Auth
bun add @magnet-cms/adapter-auth-supabase @supabase/supabase-js

# Storage
bun add @magnet-cms/adapter-storage-supabase @supabase/supabase-js

# Vault (optional)
bun add @magnet-cms/adapter-vault-supabase @supabase/supabase-js
```

## Supabase Auth Strategy

The Supabase auth strategy delegates user authentication to Supabase Auth. This lets you use Supabase's built-in features: social login providers, magic links, multi-factor authentication, and row-level security.

### Configuration

```typescript
import { DrizzleDatabaseAdapter } from '@magnet-cms/adapter-db-drizzle'
import { SupabaseAuthStrategy } from '@magnet-cms/adapter-auth-supabase'
import { MagnetModule } from '@magnet-cms/core'
import { Module } from '@nestjs/common'
import { ConfigModule } from '@nestjs/config'

@Module({
  imports: [
    ConfigModule.forRoot({ isGlobal: true }),
    MagnetModule.forRoot(
      [
        DrizzleDatabaseAdapter.forRoot({ dialect: 'postgresql', driver: 'pg' }),
        SupabaseAuthStrategy.forRoot(),
      ],
      { admin: true },
    ),
  ],
})
export class AppModule {}
```

Environment variables (`SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_KEY`, etc.) are auto-resolved from `process.env`. Pass explicit config to override.

### Auth Configuration Options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `supabaseUrl` | `string` | Yes | Your Supabase project URL (`https://xxx.supabase.co`) |
| `supabaseKey` | `string` | Yes | Supabase anon/public key |
| `supabaseServiceKey` | `string` | No | Service role key — required for `listUsers()` and user management |
| `defaultRole` | `string` | No | Default role assigned to new users (default: `'user'`) |

### How Auth Works

The Supabase auth strategy integrates with Magnet's auth module:

| Operation | Behavior |
|-----------|----------|
| `POST /auth/login` | Calls `supabase.auth.signInWithPassword()` |
| `POST /auth/register` | Calls `supabase.auth.signUp()` |
| `GET /auth/me` | Validates JWT against Supabase, returns user data |
| Token refresh | Calls `supabase.auth.refreshSession()` |
| Logout | Calls `supabase.auth.signOut()` |

### Environment Variables

```bash
# Supabase project settings (Dashboard → Settings → API)
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_KEY=your-service-role-key  # Optional

# Database connection (Supabase PostgreSQL via Drizzle)
DATABASE_URL=postgresql://postgres:password@db.your-project.supabase.co:5432/postgres
```

## Supabase Storage Adapter

The storage adapter stores media uploads in Supabase Storage buckets.

### Configuration

```typescript
import { DrizzleDatabaseAdapter } from '@magnet-cms/adapter-db-drizzle'
import { SupabaseStorageAdapter } from '@magnet-cms/adapter-storage-supabase'
import { MagnetModule } from '@magnet-cms/core'
import { Module } from '@nestjs/common'

@Module({
  imports: [
    MagnetModule.forRoot(
      [
        DrizzleDatabaseAdapter.forRoot({ dialect: 'postgresql', driver: 'pg' }),
        SupabaseStorageAdapter.forRoot({ bucket: 'media' }),
      ],
      { admin: true },
    ),
  ],
})
export class AppModule {}
```

Environment variables (`SUPABASE_URL`, `SUPABASE_SERVICE_KEY`, `SUPABASE_STORAGE_BUCKET`) are auto-resolved from `process.env`.

### Storage Adapter Options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `supabaseUrl` | `string` | Yes | Your Supabase project URL |
| `supabaseKey` | `string` | Yes | Service role key (needed for write operations) |
| `bucket` | `string` | Yes | Storage bucket name (create it in Supabase dashboard) |

### Creating the Storage Bucket

Before using the storage adapter, create a bucket in your Supabase dashboard:

1. Go to **Storage** in your Supabase project
2. Click **New bucket**
3. Name it `media` (or match your `bucket` config)
4. Set **Public** if you want files to be publicly accessible

Or use the Supabase CLI:

```bash
supabase storage create-bucket media --public
```

## Supabase Vault Adapter

The vault adapter uses Supabase Vault (pgsodium) for encrypted secrets management.

### Configuration

```typescript
import { SupabaseVaultAdapter } from '@magnet-cms/adapter-vault-supabase'

MagnetModule.forRoot(
  [
    DrizzleDatabaseAdapter.forRoot({ dialect: 'postgresql', driver: 'pg' }),
    SupabaseVaultAdapter.forRoot(),
  ],
  { admin: true },
)
```

## Combined Setup: Drizzle + Supabase

The most common setup uses Drizzle for the database (connecting to Supabase PostgreSQL) and Supabase for auth, storage, and vault:

```typescript
import { SupabaseAuthStrategy } from '@magnet-cms/adapter-auth-supabase'
import { DrizzleDatabaseAdapter } from '@magnet-cms/adapter-db-drizzle'
import { SupabaseStorageAdapter } from '@magnet-cms/adapter-storage-supabase'
import { SupabaseVaultAdapter } from '@magnet-cms/adapter-vault-supabase'
import { MagnetModule } from '@magnet-cms/core'
import { PlaygroundPlugin } from '@magnet-cms/plugin-playground'
import { Module } from '@nestjs/common'
import { ConfigModule } from '@nestjs/config'

@Module({
  imports: [
    ConfigModule.forRoot({ isGlobal: true }),
    MagnetModule.forRoot(
      [
        DrizzleDatabaseAdapter.forRoot({
          dialect: 'postgresql',
          driver: 'pg',
          migrations: { mode: 'auto', directory: './migrations' },
        }),
        SupabaseAuthStrategy.forRoot(),
        SupabaseStorageAdapter.forRoot(),
        SupabaseVaultAdapter.forRoot(),
        PlaygroundPlugin.forRoot(),
      ],
      { admin: true },
    ),
  ],
})
export class AppModule {}
```

Required environment variables are validated at startup. If any are missing, the app exits with a clear error listing all missing vars.

## Local Development with Docker

For local development, you can run a self-hosted Supabase stack using Docker. A ready-to-use configuration is in [`apps/examples/drizzle-supabase/docker`](https://github.com/magnet-cms/magnet/tree/main/apps/examples/drizzle-supabase/docker):

```bash
cd apps/examples/drizzle-supabase
cp .env.example .env
bun run docker:up
```

Or manually:

```bash
docker compose -f docker/docker-compose.yml --env-file .env up -d
```

For a clean start (fresh volumes):

```bash
docker compose -f docker/docker-compose.yml down -v
bun run docker:up
```

The local Supabase stack starts:

- PostgreSQL on port 5432
- Supabase API (Kong) on port 8000
- Supabase Studio on port **3010** (host)
- Storage (internal; accessed via Kong on port 8000)

## Example Application

A complete working example using Drizzle + Supabase is available in [`apps/examples/drizzle-supabase`](https://github.com/magnet-cms/magnet/tree/main/apps/examples/drizzle-supabase).

## What This Adapter Does NOT Provide

- **Database adapter** — Use `@magnet-cms/adapter-db-drizzle` for database connectivity
- **Row-level security (RLS)** — Configure RLS policies directly in Supabase dashboard
- **Realtime subscriptions** — Use the Supabase JS client directly in your frontend
- **Edge functions** — Deploy separately via Supabase CLI
