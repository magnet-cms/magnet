# Database Adapters

Magnet supports multiple databases through an adapter pattern. The database adapter is the only required provider ‚Äî every other adapter is optional.

## The DatabaseAdapter Contract

All database adapters implement the `DatabaseAdapter` abstract class from `@magnet-cms/common`. This interface is the only surface area that core modules depend on:

```typescript
abstract class DatabaseAdapter {
  abstract connect(config: DBConfig): DynamicModule
  abstract forFeature(schemas: Type | Type[]): DynamicModule
  abstract model<T>(modelInstance: any): any
  abstract token(schema: string): string
  abstract supports(capability: string): boolean
  abstract getCapabilities(): string[]
}
```

Core services like `ContentService` receive a `Model<T>` instance through NestJS injection and call `find`, `findOne`, `create`, `update`, `delete`, `findMany`, `findById` ‚Äî adapter-specific implementation details are invisible to them.

## Adapter Comparison

| Feature | Mongoose | Drizzle |
|---------|---------|---------|
| Databases | MongoDB | PostgreSQL, MySQL, SQLite |
| Schema definition | Decorator-driven | Decorator-driven |
| Migrations | Not applicable | First-class CLI support |
| Query builder | Mongoose queries | Drizzle ORM |
| Full-text search | MongoDB text indexes | Database-native |
| Transactions | Supported | Supported |

## Auto-Detection

Importing an adapter package registers the adapter type as a side effect:

```typescript
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'
// ‚Üê setDatabaseAdapter('mongoose') is called automatically
```

This means `@Schema()` decorators know which database format to use without any manual wiring. You never configure the adapter type in multiple places.

## Injection Tokens

The database module exposes typed injection tokens so you can inject model instances in your own services:

```typescript
import { getModelToken } from '@magnet-cms/common'
import { Inject } from '@nestjs/common'

@Injectable()
class CatsService {
  constructor(
    @Inject(getModelToken('Cat')) private catModel: Model<Cat>,
  ) {}
}
```

## Custom Adapters

To support a new database, subclass `DatabaseAdapter` from `@magnet-cms/common`, implement all abstract methods, and return a `DatabaseMagnetProvider` from a static `forRoot()` factory. The existing Mongoose and Drizzle adapter sources are the reference implementation.
