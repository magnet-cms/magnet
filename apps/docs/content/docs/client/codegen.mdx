`@magnet-cms/client` ships with a `magnet-client` CLI that generates TypeScript type definitions from your Magnet CMS server's OpenAPI spec. The generated types power full autocomplete and type checking in the client SDK.

## How It Works

1. Your Magnet server exposes an OpenAPI 3.0 spec at `/oas.json`
2. `magnet-client generate` downloads the spec and runs `openapi-typescript` on it
3. A `.d.ts` file is written to your project with all API paths and schemas typed
4. You pass the generated `paths` type to `createMagnetClient<paths>()` for end-to-end type safety

## Installation

```bash
bun add @magnet-cms/client
# or
npm install @magnet-cms/client
```

## Usage

### Generate from a running server

Start your Magnet app, then run:

```bash
npx magnet-client generate --url http://localhost:3000/oas.json -o ./src/magnet.d.ts
```

### Generate from a local spec file

If you have a downloaded or checked-in `openapi.json`:

```bash
npx magnet-client generate --input ./openapi.json -o ./src/magnet.d.ts
```

### Options

| Flag | Alias | Default | Description |
|------|-------|---------|-------------|
| `--url <url>` | | | URL of the OpenAPI spec (e.g., `http://localhost:3000/oas.json`) |
| `--input <path>` | | | Path to a local OpenAPI JSON/YAML file |
| `--output <path>` | `-o` | `./magnet.d.ts` | Output file path for the generated types |

Either `--url` or `--input` is required.

## Workflow Integration

### package.json script

Add a `generate` script to regenerate types when the server schema changes:

```json
{
  "scripts": {
    "generate:types": "magnet-client generate --url http://localhost:3000/oas.json -o ./src/magnet.d.ts"
  }
}
```

Run after adding or modifying content schemas:

```bash
bun run generate:types
```

### CI/CD

You can also check in the generated file and regenerate it in CI when deploying a new version of your CMS:

```yaml
- name: Regenerate Magnet types
  run: bun run generate:types
```

## Using Generated Types

After generating `src/magnet.d.ts`, import the `paths` type and pass it to the client factory:

```typescript
import { createMagnetClient } from '@magnet-cms/client'
import type { paths } from './magnet.d.ts'

const client = createMagnetClient<paths>({
  baseUrl: 'http://localhost:3000',
  token: process.env.MAGNET_TOKEN,
})

// Full autocomplete on paths, methods, query params, and response shapes
const { data } = await client.raw.GET('/content/{schema}', {
  params: { path: { schema: 'post' } },
})
```

## What Gets Generated

The generated file contains TypeScript interfaces for:

- All API paths (`paths`) with their HTTP methods, parameters, and response bodies
- All request body shapes
- All response schemas
- Component schemas for each content type

Example excerpt for a `Post` content type:

```typescript
export interface paths {
  '/content/{schema}': {
    get: {
      parameters: {
        path: { schema: string }
        query?: { page?: number; limit?: number; status?: string }
      }
      responses: {
        200: {
          content: {
            'application/json': {
              items: components['schemas']['Post'][]
              total: number
            }
          }
        }
      }
    }
    // ...
  }
}

export interface components {
  schemas: {
    Post: {
      _id: string
      title: string
      body?: string
      publishedAt?: string
    }
  }
}
```

## Troubleshooting

**`Either --url or --input must be provided`**
You must specify one of `--url` or `--input`.

**`ECONNREFUSED` when using `--url`**
Make sure your Magnet server is running and accessible at the specified URL.

**Generated file has no content schemas**
Ensure the Discovery module is active (it is by default). Content schemas are dynamically added to the spec when the server starts.
