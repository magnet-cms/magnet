# Admin UI Internationalization

The Magnet Admin UI uses [react-intl](https://formatjs.github.io/docs/react-intl/) (FormatJS) for internationalization. All user-facing strings are stored in message catalogs that can be translated into multiple languages.

## Supported Locales

- `en` — English (default)
- `pt-BR` — Portuguese (Brazil)
- `es` — Spanish

## How It Works

### Message Catalogs

All translatable strings live in JSON files under `packages/client/admin/src/i18n/messages/`:

```
src/i18n/messages/
  en.json        # English (source of truth)
  pt-BR.json     # Portuguese (Brazil)
  es.json        # Spanish
```

Each file maps message IDs to translated strings:

```json
{
  "auth.login.title": "Welcome back",
  "common.actions.save": "Save"
}
```

### Message ID Convention

Message IDs follow `{feature}.{section}.{element}`:

- `auth.login.title` — Auth feature, login section, title element
- `common.actions.save` — Common namespace, save button

### Using Translations in Components

```tsx
import { useAppIntl } from '~/i18n'

export function MyComponent() {
  const intl = useAppIntl()

  return (
    <h1>
      {intl.formatMessage({
        id: 'myFeature.title',
        defaultMessage: 'My Feature',
      })}
    </h1>
  )
}
```

## Adding a New Translatable String

1. Add the key and English text to `src/i18n/messages/en.json`
2. Add the same key to `pt-BR.json` and `es.json`
3. Use `intl.formatMessage({ id: 'your.key', defaultMessage: 'English text' })` in the component

## Adding a New Locale

1. Create a new JSON file in `src/i18n/messages/` (e.g., `fr.json`)
2. Copy all keys from `en.json` and translate the values
3. Add the locale to `SUPPORTED_LOCALES` in `src/i18n/provider.tsx`
4. Add a label in `LanguageSettingsCard.tsx`

## Library Usage

When embedding `MagnetAdmin` in a host application, control the locale via props:

```tsx
import { MagnetAdmin } from '@magnet-cms/admin'

function App() {
  return (
    <MagnetAdmin
      locale="pt-BR"
      messages={myCustomMessages}
    />
  )
}
```

## Locale Detection Order

1. `locale` prop on `MagnetAdmin` (if provided)
2. Value stored in `localStorage` (key: `magnet-admin-locale`)
3. Browser language (`navigator.language`)
4. Fallback to `en`

Non-English locale bundles are lazy-loaded to keep the main bundle size small.
