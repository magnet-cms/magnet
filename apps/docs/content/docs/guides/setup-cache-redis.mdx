# Setup Cache (Redis)

Add a Redis-backed cache to Magnet CMS. Compatible with Redis 6+ and Dragonfly.

Without a cache adapter, Magnet uses in-process memory caching (TTL-based maps). Redis cache is shared across multiple instances and survives restarts.

## Prerequisites

- A Redis server (local Docker, Redis Cloud, Upstash, or Dragonfly)
- NestJS app with `@magnet-cms/core` and a database adapter installed

## 1. Install

```bash
bun add @magnet-cms/adapter-cache-redis ioredis
```

## 2. Set Environment Variables

```bash
# .env
REDIS_URL=redis://localhost:6379
# Or, if using a dedicated cache instance:
CACHE_REDIS_URL=redis://cache.example.com:6379
```

`CACHE_REDIS_URL` takes precedence over `REDIS_URL` if both are set.

## 3. Register the Adapter

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'
import { RedisCacheAdapter } from '@magnet-cms/adapter-cache-redis'

@Module({
  imports: [
    MagnetModule.forRoot([
      MongooseDatabaseAdapter.forRoot(),
      RedisCacheAdapter.forRoot(),
    ]),
  ],
})
export class AppModule {}
```

With explicit config:

```typescript
RedisCacheAdapter.forRoot({
  url: process.env.REDIS_URL,
  keyPrefix: 'myapp:',
})
```

## 4. Local Development with Docker

```yaml
# docker-compose.yml
services:
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
```

```bash
docker compose up -d redis
```

## 5. Verify

Check the application logs after startup:

```bash
bun run start:dev
# Look for: "Redis cache connected"
```
