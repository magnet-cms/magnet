# Setup Playground

The Playground plugin adds a visual schema builder to the admin UI. Create content schemas without writing TypeScript — the plugin generates fully typed schema, controller, service, and DTO files.

## 1. Install

```bash
bun add @magnet-cms/plugin-playground
```

## 2. Register

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'
import { PlaygroundPlugin } from '@magnet-cms/plugin-playground'

@Module({
  imports: [
    MagnetModule.forRoot([
      MongooseDatabaseAdapter.forRoot(),
      PlaygroundPlugin.forRoot({
        modulesPath: './src/modules',
      }),
    ]),
  ],
})
export class AppModule {}
```

The `modulesPath` option sets where generated schema files are written. Default: `./src/modules`.

## 3. Use the Playground

1. Start the application and navigate to the admin UI
2. Click **Playground** in the sidebar
3. Click **Create New**
4. Enter a schema name and add fields
5. Click **Save** — the plugin generates TypeScript files in `modulesPath`

## 4. Use Generated Schemas

After saving a schema, import the generated module in your `AppModule`:

```typescript
// Generated by Playground
import { ArticleModule } from './modules/article/article.module'

@Module({
  imports: [
    MagnetModule.forRoot([...]),
    ArticleModule,
  ],
})
export class AppModule {}
```

Restart the server — the new content type appears in the content sidebar.

## Note: Development Only

The Playground is best used in development. In production, use the generated TypeScript files directly and remove the plugin:

```typescript
if (process.env.NODE_ENV === 'development') {
  providers.push(PlaygroundPlugin.forRoot())
}
```
