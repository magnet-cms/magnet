# Setup S3 Storage

Route Magnet media uploads to Amazon S3 or any S3-compatible storage (MinIO, Backblaze B2, etc.).

## Prerequisites

- An S3 bucket with the appropriate IAM permissions
- NestJS app with `@magnet-cms/core` and a database adapter installed

## 1. Install

```bash
bun add @magnet-cms/adapter-storage-s3 @aws-sdk/client-s3 @aws-sdk/s3-request-presigner
```

## 2. Set Environment Variables

```bash
# .env
S3_BUCKET=my-media-bucket
S3_REGION=us-east-1
S3_ACCESS_KEY_ID=AKIA...
S3_SECRET_ACCESS_KEY=...
S3_PUBLIC_URL=https://my-media-bucket.s3.us-east-1.amazonaws.com
# Optional: custom endpoint for S3-compatible APIs
# S3_ENDPOINT=https://s3.us-west-000.backblazeb2.com
```

## 3. Register the Adapter

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'
import { S3StorageAdapter } from '@magnet-cms/adapter-storage-s3'

@Module({
  imports: [
    MagnetModule.forRoot([
      MongooseDatabaseAdapter.forRoot(),
      S3StorageAdapter.forRoot({
        bucket: process.env.S3_BUCKET,
        region: process.env.S3_REGION ?? 'us-east-1',
        publicUrl: process.env.S3_PUBLIC_URL,
      }),
    ]),
  ],
})
export class AppModule {}
```

## 4. Configure Bucket Permissions

Your IAM user needs these permissions on the bucket:

```json
{
  "Effect": "Allow",
  "Action": [
    "s3:PutObject",
    "s3:GetObject",
    "s3:DeleteObject",
    "s3:ListBucket"
  ],
  "Resource": [
    "arn:aws:s3:::my-media-bucket",
    "arn:aws:s3:::my-media-bucket/*"
  ]
}
```

## 5. Verify

Upload a file via the admin UI Media Library and confirm it appears in your S3 bucket.

## S3-Compatible APIs (MinIO)

For MinIO or other S3-compatible APIs, set `S3_ENDPOINT`:

```typescript
S3StorageAdapter.forRoot({
  bucket: 'media',
  region: 'us-east-1',
  endpoint: 'http://localhost:9000',
  publicUrl: 'http://localhost:9000/media',
})
```
