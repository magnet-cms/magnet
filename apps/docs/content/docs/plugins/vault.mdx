## Overview

The Vault module provides encrypted secrets management in Magnet CMS. It is built into `@magnet-cms/core` and supports three adapters:

- **DB** (default) — Stores encrypted secrets in your database (requires `VAULT_MASTER_KEY`)
- **HashiCorp** — Uses [HashiCorp Vault](https://www.vaultproject.io/) for centralized secrets
- **Supabase** — Uses Supabase Vault (pgsodium) for encrypted storage

## Configuration

Configure the vault in `MagnetModule.forRoot()`:

### Built-in DB Adapter (default)

Secrets are stored encrypted in your database. Requires `VAULT_MASTER_KEY` environment variable (min 32 characters).

```typescript
MagnetModule.forRoot({
  db: { uri: process.env.MONGODB_URI || '...' },
  jwt: { secret: process.env.JWT_SECRET || '...' },
  vault: { adapter: 'db' },
})
```

```bash
# .env
VAULT_MASTER_KEY=your-master-key-with-at-least-32-characters
```

### HashiCorp Vault

```typescript
MagnetModule.forRoot({
  db: { uri: '...' },
  jwt: { secret: '...' },
  vault: {
    adapter: 'hashicorp',
    hashicorp: {
      url: process.env.VAULT_ADDR || 'http://localhost:8200',
      auth: { type: 'token', token: process.env.VAULT_TOKEN || '' },
    },
  },
})
```

Install the adapter:

```bash
bun add @magnet-cms/adapter-vault-hashicorp
```

### Supabase Vault

```typescript
MagnetModule.forRoot({
  db: { connectionString: process.env.DATABASE_URL, dialect: 'postgresql', driver: 'pg' },
  jwt: { secret: process.env.JWT_SECRET || '' },
  vault: {
    adapter: 'supabase',
    supabase: {
      supabaseUrl: process.env.SUPABASE_URL || '',
      supabaseServiceKey: process.env.SUPABASE_SERVICE_KEY || '',
    },
  },
})
```

Install the adapter:

```bash
bun add @magnet-cms/adapter-vault-supabase
```

## Runtime Usage

Inject `VaultService` from `@magnet-cms/core` for secret resolution:

```typescript
import { Injectable } from '@nestjs/common'
import { VaultService } from '@magnet-cms/core'

@Injectable()
export class MyService {
  constructor(private readonly vault: VaultService) {}

  async getApiKey(): Promise<string | null> {
    return this.vault.get('api-keys')
  }

  async storeSecret(key: string, value: string, description?: string): Promise<void> {
    await this.vault.set(key, value, description)
  }

  async refreshCachedSecret(key: string): Promise<void> {
    await this.vault.refresh(key)
  }
}
```

### VaultService Methods

| Method | Description |
|--------|-------------|
| `get(key)` | Retrieve a secret (cached). Returns `string \| null` |
| `set(key, value, description?)` | Store or update a secret |
| `delete(key)` | Delete a secret |
| `list(prefix?)` | List secrets with metadata |
| `refresh(key)` | Clear cache and re-fetch a secret |
| `healthCheck()` | Test vault backend connectivity |
| `getAdapterType()` | Get current adapter (`'db' \| 'hashicorp' \| 'supabase'`) |

## Environment Variables

| Variable | Adapter | Description |
|----------|---------|-------------|
| `VAULT_MASTER_KEY` | DB | Master key for encryption (min 32 chars) |
| `VAULT_ADDR` | HashiCorp | Vault server URL |
| `VAULT_TOKEN` | HashiCorp | Authentication token |
| `SUPABASE_URL` | Supabase | Supabase project URL |
| `SUPABASE_SERVICE_KEY` | Supabase | Service role key |

## Local Development

### DB Adapter (simplest)

Set `VAULT_MASTER_KEY` in `.env` and use the default DB adapter. Secrets are stored in your database.

### HashiCorp Vault dev server

```bash
vault server -dev -dev-root-token-id="dev-token"
export VAULT_ADDR=http://127.0.0.1:8200
export VAULT_TOKEN=dev-token
```

### Supabase Vault

Use the [Supabase adapter](/docs/adapters/supabase) with the local Docker stack. The vault uses Supabase's pgsodium extension.
