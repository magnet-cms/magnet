The Database Module manages database connections and provides a unified interface for different database adapters in Magnet CMS.

## Overview

The Database Module provides:

- **Adapter Abstraction**: Unified API across different databases
- **Schema Registration**: Register content schemas with the database
- **i18n Support**: Multi-locale content storage
- **Auto-Detection**: Automatically detects installed adapter

## Supported Databases

| Database | Adapter Package | Description |
|----------|-----------------|-------------|
| MongoDB | `@magnet-cms/adapter-db-mongoose` | Full-featured MongoDB adapter |
| PostgreSQL | `@magnet-cms/adapter-db-drizzle` | Via Drizzle ORM |
| MySQL | `@magnet-cms/adapter-db-drizzle` | Via Drizzle ORM |
| SQLite | `@magnet-cms/adapter-db-drizzle` | Via Drizzle ORM |
| Supabase | `@magnet-cms/adapter-db-drizzle` + auth/storage adapters | PostgreSQL via Drizzle; auth/storage via Supabase |

## Configuration

### MongoDB (Mongoose)

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'

@Module({
  imports: [
    MagnetModule.forRoot([
      MongooseDatabaseAdapter.forRoot(),
    ], { admin: true }),
  ],
})
export class AppModule {}
```

`MongooseDatabaseAdapter.forRoot()` auto-resolves `MONGODB_URI` from `process.env`.

### PostgreSQL (Drizzle)

```typescript
import { DrizzleDatabaseAdapter } from '@magnet-cms/adapter-db-drizzle'

MagnetModule.forRoot([
  DrizzleDatabaseAdapter.forRoot({
    dialect: 'postgresql',
    driver: 'neon',  // or 'pg'
  }),
], { admin: true })
```

### MySQL (Drizzle)

```typescript
MagnetModule.forRoot([
  DrizzleDatabaseAdapter.forRoot({
    dialect: 'mysql',
    driver: 'mysql2',
  }),
], { admin: true })
```

### SQLite (Drizzle)

```typescript
MagnetModule.forRoot([
  DrizzleDatabaseAdapter.forRoot({
    connectionString: './data/database.sqlite',
    dialect: 'sqlite',
    driver: 'better-sqlite3',
  }),
], { admin: true })
```

`DrizzleDatabaseAdapter.forRoot()` auto-resolves `DATABASE_URL` from `process.env` for the connection string.

## Registering Schemas

Register your content schemas in your module:

```typescript
import { Module } from '@nestjs/common'
import { DatabaseModule } from '@magnet-cms/core'
import { Cat } from './cat.schema'
import { CatsService } from './cats.service'
import { CatsController } from './cats.controller'

@Module({
  imports: [
    DatabaseModule.forFeature([Cat]),
  ],
  controllers: [CatsController],
  providers: [CatsService],
})
export class CatsModule {}
```

Multiple schemas can be registered at once:

```typescript
DatabaseModule.forFeature([Cat, Dog, Bird])
```

## Database Adapter Interface

All adapters implement the `DatabaseAdapter` interface:

```typescript
abstract class DatabaseAdapter {
  // Initialize database connection
  abstract connect(config: DBConfig): DynamicModule

  // Register schemas with the database
  abstract forFeature(schemas: Type | Type[]): DynamicModule

  // Get model wrapper for a schema
  abstract model<T>(modelInstance: any): any

  // Generate injection token for a schema
  abstract token(schema: string): string
}
```

## Auto-Detection

The database adapter is automatically registered when you import an adapter package. Each adapter's `index.ts` calls `setDatabaseAdapter()` as a side effect, so `@Schema()` decorators resolve correctly without any manual setup.

```typescript
// Importing the adapter package auto-registers the adapter type
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'
// → setDatabaseAdapter('mongoose') is called automatically

import { DrizzleDatabaseAdapter } from '@magnet-cms/adapter-db-drizzle'
// → setDatabaseAdapter('drizzle') is called automatically
```

No `setDatabaseAdapter()` call is needed in your application code.

## i18n Configuration

Configure internationalization in the global options:

```typescript
MagnetModule.forRoot([
  MongooseDatabaseAdapter.forRoot(),
], {
  admin: true,
  internationalization: {
    locales: ['en', 'pt-BR', 'es'],
    defaultLocale: 'en',
  },
})
```

This enables:
- Locale-specific content versions
- Draft/published status per locale
- Locale fallback queries

## Model Operations

After registering a schema, inject the model in your service:

```typescript
import { Injectable } from '@nestjs/common'
import { InjectModel, Model } from '@magnet-cms/common'
import { Cat } from './cat.schema'

@Injectable()
export class CatsService {
  constructor(
    @InjectModel(Cat)
    private model: Model<Cat>,
  ) {}

  // Use the model for database operations
  async findAll() {
    return this.model.find()
  }

  async findByLocale(locale: string) {
    return this.model.locale(locale).find()
  }
}
```

## Troubleshooting

### Adapter Not Found

```
Error: Database adapter not found
```

Install the appropriate adapter:

```bash
# For MongoDB
bun add @magnet-cms/adapter-db-mongoose mongoose

# For PostgreSQL/MySQL/SQLite
bun add @magnet-cms/adapter-db-drizzle drizzle-orm pg
```

### Connection Errors

1. Verify database is running
2. Check connection string format
3. Ensure network access (for cloud databases)
4. Check authentication credentials
