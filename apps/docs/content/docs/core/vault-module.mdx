## Overview

The Vault module provides encrypted secrets management in Magnet CMS. It is built into `@magnet-cms/core` and supports three backends:

- **DB** (default) — Stores encrypted secrets in your database (requires `VAULT_MASTER_KEY`)
- **HashiCorp** — Uses [HashiCorp Vault](https://www.vaultproject.io/) KV v2
- **Supabase** — Uses Supabase Vault (pgsodium) via `@magnet-cms/adapter-vault-supabase`

Register optional vault providers in the same **provider array** as your database adapter (`MagnetModule.forRoot([...])`). If you omit a vault provider, Magnet uses the built-in **DB** vault adapter automatically.

## Built-in DB adapter (default)

Secrets are stored encrypted in your database. Set `VAULT_MASTER_KEY` (minimum 32 characters).

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'

@Module({
  imports: [
    MagnetModule.forRoot([MongooseDatabaseAdapter.forRoot()]),
  ],
})
export class AppModule {}
```

```bash
# .env
VAULT_MASTER_KEY=your-master-key-with-at-least-32-characters
```

## HashiCorp Vault

Add `HashiCorpVaultAdapter.forRoot()` to the provider array after your database adapter.

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'
import { HashiCorpVaultAdapter } from '@magnet-cms/adapter-vault-hashicorp'

@Module({
  imports: [
    MagnetModule.forRoot([
      MongooseDatabaseAdapter.forRoot(),
      HashiCorpVaultAdapter.forRoot({
        url: process.env.VAULT_ADDR || 'http://localhost:8200',
        mountPath: 'secret',
        auth: { type: 'token', token: process.env.VAULT_TOKEN || '' },
      }),
    ]),
  ],
})
export class AppModule {}
```

```bash
bun add @magnet-cms/adapter-vault-hashicorp
```

See [HashiCorp Vault adapter](/docs/adapters/hashicorp-vault) for environment variables and KV v2 paths.

## Supabase Vault

Uses a direct PostgreSQL connection to `vault.decrypted_secrets` (see [Supabase adapters](/docs/adapters/supabase) for the full stack).

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { DrizzleDatabaseAdapter } from '@magnet-cms/adapter-db-drizzle'
import { SupabaseVaultAdapter } from '@magnet-cms/adapter-vault-supabase'

@Module({
  imports: [
    MagnetModule.forRoot([
      DrizzleDatabaseAdapter.forRoot({
        dialect: 'postgresql',
        driver: 'pg',
      }),
      SupabaseVaultAdapter.forRoot(),
    ]),
  ],
})
export class AppModule {}
```

```bash
bun add @magnet-cms/adapter-vault-supabase
```

`SupabaseVaultAdapter` resolves `DATABASE_URL` by default (same database as Drizzle).

## Runtime usage

Inject `VaultService` from `@magnet-cms/core`:

```typescript
import { Injectable } from '@nestjs/common'
import { VaultService } from '@magnet-cms/core'

@Injectable()
export class MyService {
  constructor(private readonly vault: VaultService) {}

  async getApiKey(): Promise<string | null> {
    return this.vault.get('api-keys')
  }

  async storeSecret(key: string, value: string, description?: string): Promise<void> {
    await this.vault.set(key, value, description)
  }

  async refreshCachedSecret(key: string): Promise<void> {
    await this.vault.refresh(key)
  }
}
```

### VaultService methods

| Method | Description |
|--------|-------------|
| `get(key)` | Retrieve a secret (cached). Returns `string \| null` |
| `set(key, value, description?)` | Store or update a secret |
| `delete(key)` | Delete a secret |
| `list(prefix?)` | List secrets with metadata |
| `refresh(key)` | Clear cache and re-fetch a secret |
| `healthCheck()` | Test vault backend connectivity |
| `getAdapterType()` | Current adapter (`'db' \| 'hashicorp' \| 'supabase'`) |

## Environment variables

| Variable | Backend | Description |
|----------|---------|-------------|
| `VAULT_MASTER_KEY` | DB | Master key for encryption (min 32 chars) |
| `VAULT_ADDR` | HashiCorp | Vault server URL |
| `VAULT_TOKEN` | HashiCorp | Token auth (or use AppRole via adapter config) |
| `DATABASE_URL` | Supabase adapter | PostgreSQL connection string for vault queries |

## Local development

### DB adapter

Set `VAULT_MASTER_KEY` in `.env` and use the default adapter.

### HashiCorp dev server

```bash
vault server -dev -dev-root-token-id="dev-token"
export VAULT_ADDR=http://127.0.0.1:8200
export VAULT_TOKEN=dev-token
```

### Supabase Vault

Use the [Supabase](/docs/adapters/supabase) guide and local Docker stack. The vault relies on Supabase’s pgsodium / `vault` schema.
