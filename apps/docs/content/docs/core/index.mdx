# Core Module

The `@magnet-cms/core` package bootstraps the CMS: HTTP APIs, admin serving, database abstraction, plugins, and cross-cutting services.

## Bootstrapping

```typescript
import { Module } from '@nestjs/common'
import { MagnetModule } from '@magnet-cms/core'
import { MongooseDatabaseAdapter } from '@magnet-cms/adapter-db-mongoose'

@Module({
  imports: [
    MagnetModule.forRoot([MongooseDatabaseAdapter.forRoot()]),
  ],
})
export class AppModule {}
```

Pass **providers** (database, auth, storage, vault, email, cache, GraphQL, plugins) in the first argument. Optional **global** options are the second argument when using the array form (`jwt`, `rbac`, `internationalization`, `playground`, `email.layout`, …). The admin UI is enabled by default; use `admin: false` for API-only apps. See [Configuration](/docs/getting-started/configuration).

Each adapter or plugin exposes `static forRoot()` and declares environment variables for startup validation.

## Built-in areas

| Area | Role |
|------|------|
| [Content](/docs/core/content-module) | Schemas, CRUD, drafts, locales, versions |
| [Auth](/docs/core/auth-module) | Strategies (JWT, Supabase, Clerk, …), sessions |
| [API keys](/docs/core/api-keys-module) | Programmatic access |
| [RBAC](/docs/core/rbac-module) | Roles and permissions |
| [Admin API](/docs/core/admin-module) | Endpoints backing the admin UI |
| [Database](/docs/core/database-module) | Adapter registration and schemas |
| [Email](/docs/core/email-module) | Transactional email adapters |
| [Cache](/docs/core/cache-module) | Memory or Redis-backed cache |
| [Vault](/docs/core/vault-module) | Encrypted secrets |
| [Settings](/docs/core/settings-module) | Typed settings and `/settings` API |
| [Storage](/docs/core/storage-module) | Object storage adapters |
| [Notifications](/docs/core/notifications-module) | Platform and email channels |
| [Webhooks](/docs/core/webhooks-module) | Outbound webhook delivery |

Plugins add more routes and sidebar entries; see [Plugins](/docs/plugins).
